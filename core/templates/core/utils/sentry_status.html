To test that error reporting is working correctly, you can:

What's Being Captured

When an error occurs, Sentry captures:

  • Exception details and stack trace
  • Request information (URL, headers, etc.)
  • User context (if the user is logged in)
  • Environment information (Django version, Python version, etc.)
